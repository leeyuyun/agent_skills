# Style Presets Reference

Use this file to confirm and apply PPTX style choices.

## Selection Protocol (Required For New Decks)

Use a three-axis confirmation to avoid ambiguity:

1. `Primary Visual Style` (choose one from categories A-F)
2. `Layout Grammar` (choose one or more from category G)
3. `Motion Style` (choose one from category H if animation is requested)

If the user does not choose, default to:
- `A1 企業極簡（Corporate Minimal）`
- `G13 目錄章節化（Sectioned / Book-like）`
- `H1 乾淨商務（少轉場、弱動作）`

## User Prompt Template

Use this template when style is unspecified:

`在開始製作前，請先選風格：`
`A-F 先選 1 個主視覺風格，G 選版面語法，H 選動態風格（若需要動畫）。`
`若你不指定，我會用 A1 + G13 + H1。`

---

## A. 商務與管理（Corporate / Executive）

- `A1 企業極簡（Corporate Minimal）`: 清爽留白、保守配色、高可讀性。
- `A2 董事會/高管簡報（Board / Executive）`: 高密度重點、結論前置、圖表克制。
- `A3 顧問風（Consulting：MECE、金字塔、框線）`: 結構化論證、框線模組、邏輯樹。
- `A4 金融投資風（Finance）`: 表格與對比主導、低飽和安全色、強調風險揭露。
- `A5 年報/ESG 報告風（Report）`: 章節秩序清楚、圖表比重高、文字規範化。
- `A6 KPI 儀表板風（Dashboard）`: 指標卡、趨勢線、同比/環比視覺一致。
- `A7 內訓教材風（Training）`: 步驟化教學、圖示化重點、可快速掃讀。

## B. 新創與募資（Startup / Pitch）

- `B1 Pitch Deck 經典風（Story -> Traction -> Ask）`: 投資人熟悉的敘事節奏。
- `B2 產品敘事風（Problem-Solution-Why Now）`: 痛點到解法的遞進說服。
- `B3 增長/數據驅動風（Growth）`: 曲線、漏斗、cohort 等成長證據優先。
- `B4 Demo Day 舞台風`: 大字強節奏、少字高衝擊。
- `B5 One-metric 重點風`: 每頁只打 1 個關鍵指標與解讀。

## C. 科技與工程（Tech / Engineering）

- `C1 科技極簡白底（Clean Tech）`: 白底高對比、系統感線條。
- `C2 深色霓虹科技（Dark Tech / Neon）`: 深色背景加亮色發光重點。
- `C3 藍白工程風（Engineering）`: 網格、線框、示意圖清楚。
- `C4 資安/網路拓撲風（Cyber / Network）`: 節點、連線、層級防線視覺化。
- `C5 產品規格書風（Spec）`: 規格表、版本欄、變更紀錄標準化。
- `C6 論文/學術報告風（Academic）`: 方法/結果分區、引用區明確。

## D. 行銷與品牌（Marketing / Brand）

- `D1 品牌手冊一致化（Brand System）`: 嚴格遵循字體、色票與品牌元件。
- `D2 雜誌編輯風（Editorial）`: 大圖與留白、精排版。
- `D3 海報式宣傳風（Poster）`: 單頁主視覺強衝擊。
- `D4 社群圖卡風（Social）`: 短句、符號、強對比，適合快速傳播。
- `D5 電商促銷風（E-commerce）`: 價格標、亮色促銷資訊、CTA 明顯。
- `D6 活動發布/Keynote 風（Launch）`: 節奏分明、大片視覺、少文字。

## E. 教育與敘事（Education / Storytelling）

- `E1 章節故事書風（Storyboard）`: 分鏡式節奏與旁白節點。
- `E2 白板教學風（Whiteboard）`: 手繪線條、逐步演進、教學導向。
- `E3 圖解百科風（Explainer）`: 資訊圖、icon、流程圖為核心。
- `E4 兒童友善風（Kid-friendly）`: 柔和色、圓角、插畫化。

## F. 視覺語言/材質質感（UI/Art Aesthetics）

- `F1 扁平化（Flat）`: 無材質、純色塊、乾淨俐落。
- `F2 Material Design`: 卡片陰影、層級分明。
- `F3 擬物（Skeuomorphic）`: 真實材質與物理感。
- `F4 玻璃擬態（Glassmorphism）`: 霧玻璃面板與半透明疊層。
- `F5 新擬態（Neumorphism）`: 柔陰影浮雕、低對比 UI 感。
- `F6 粗獷主義（Brutalism）`: 強字體高對比、刻意粗糙感。
- `F7 復古（Retro：80s/90s）`: 顆粒、霓虹、CRT 懷舊語彙。
- `F8 黑金奢華（Luxury）`: 黑底、金屬點綴、精緻字距。
- `F9 日系極簡（Japanese Minimal）`: 高留白、低飽和、節制。
- `F10 北歐極簡（Scandinavian）`: 溫潤木色、柔光與舒適感。
- `F11 工業風（Industrial）`: 灰黑金屬、格柵紋理。
- `F12 手繪插畫風（Hand-drawn）`: 手作線條與插畫主導。
- `F13 水彩/紙張質感（Watercolor / Paper texture）`: 紙感與水彩筆觸。
- `F14 3D 渲染風（3D Render）`: 擬真物件與立體光影。
- `F15 等距視角（Isometric）`: 系統架構常用的等距視角。
- `F16 線框圖（Wireframe）`: UI/產品架構低保真示意。
- `F17 拼貼（Collage）`: 剪貼紙、撕邊、多素材拼接。

## G. 版面結構風格（Layout Grammar）

- `G1 文字為主（Text-heavy）`
- `G2 圖像為主（Image-led）`
- `G3 左文右圖 / 右文左圖（Split）`
- `G4 卡片式（Cards）`
- `G5 時間軸（Timeline）`
- `G6 流程圖（Flow）`
- `G7 漏斗（Funnel）`
- `G8 象限/矩陣（2x2 / 3x3）`
- `G9 比較表（Comparison）`
- `G10 地圖/區域分佈（Map-based）`
- `G11 層級樹（Hierarchy）`
- `G12 前後對照（Before/After）`
- `G13 目錄章節化（Sectioned / Book-like）`

## H. 動畫與轉場風格（Motion Style）

- `H1 乾淨商務`: 少轉場、弱動作。
- `H2 Keynote 式節奏`: 順序揭露、視線引導。
- `H3 電影式（Cinematic）`: 推拉、景深、蒙太奇。
- `H4 UI 動效式`: 滑入、淡入、彈性曲線。
- `H5 教學逐步顯示（Build）`: 一步一露出。
- `H6 數據動態（Chart animation）`: 趨勢線生長、數字跳動。

---

## Composition Rules

- Keep one primary visual style (A-F) for the full deck unless the user explicitly asks for mixed primary styles.
- Combine at most two layout grammars (G) across one deck to avoid visual noise.
- Use motion style (H) only when requested or when delivery context requires animation.
- Always state the chosen style tuple in outputs, e.g.:
  - `Primary: A3 顧問風`
  - `Layout: G8 象限/矩陣`
  - `Motion: H1 乾淨商務`
